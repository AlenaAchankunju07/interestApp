18. Hooks in react : Hooks are predefined function in react, used to provide react component features to functional based component. to use  Hooks in component it has to call the Hooks . every Hooks name will start with 'use'  keyword
    - Rules for calling Hooks in a functional based component:
        - Import Hook in component
        - Calling the hook at the top level of the hook
        - Hooks are not Conditional
    - Different types od Hooks are avilable
        - Build in /predefined Hooks
            - useState(initial-value) : used to create state in functional based component. it will return an array of two arguments. first argumentis the 'state value' and second argument is the function name which is capable of updating the state.state updating function name generally statrt with 'set'.useState can store any type of data
            - useEffect(callbackFn,dependency) hook : used to apply side effect to the functional component.we can pass the sid effect in its first argument as call back function, we canalso pass asn optional dependency argument to decide when to apply side effects in component 
                -dependency different values
                    - no value :side eachffect will apply to the component all the time, which lead to an infinite loop 
                    - empty array : side effect will apply to the componet only at the time of loading the component in Dom
                    - [state/props/variable_name] : side effect will apply to the component not only at the time of loading of the component in DOM but also whenever the value of stste/props /variable_name given in the dependency
            - useRef(initialValue) : it is a react hook that lets you reference a value that's not needed for rendering. initialValue is optional in useRef hook, it will return an object with current key 
        - Custom Hooks : create a js file to define a specific function foe a spectal task. export the function from the file .function name must start with "use" keyword
